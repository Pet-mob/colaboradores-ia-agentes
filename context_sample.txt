# CONTEXTO TÉCNICO DETALHADO PARA DEVELOPER

## ENTIDADES DO BANCO DE DADOS

### User
Usuário da aplicação (cliente ou admin)
**Fields:** Id, Name, Email, Phone, ProfileImage, AddressId, CreatedAt
**Relationships:** Pets, Addresses, Appointments

### Pet
Animal de estimação (cachorro, gato, etc)
**Fields:** Id, Name, Type, Breed, Age, Weight, UserId, Photo, VaccineStatus
**Relationships:** User, Appointments, Services

### PetShop
Prestador de serviço (clínica, grooming, hotel, etc)
**Fields:** Id, Name, Email, Phone, Logo, Banner, AddressId, Rating, OpeningHours
**Relationships:** Address, Services, Appointments, Employees

### Service
Serviço oferecido por petshop (banho, tosa, consulta, hotel, etc)
**Fields:** Id, Name, Description, Price, Duration, PetShopId, Category, IsActive
**Relationships:** PetShop, Appointments, SubServices

### SubService
Sub-serviço ou add-on de um serviço (ex: hidratação em banho)
**Fields:** Id, Name, Price, ServiceId, Duration
**Relationships:** Service

### Appointment
Agendamento de serviço
**Fields:** Id, PetId, ServiceId, PetShopId, ScheduledDate, Status, Notes, CreatedAt
**Relationships:** Pet, Service, PetShop, User

### Address
Endereço de usuário ou petshop
**Fields:** Id, Street, Number, City, State, ZipCode, Latitude, Longitude
**Relationships:** User, PetShop

## APIs DISPONÍVEIS

### Users
- `GET /api/users` - Listar todos os usuários
- `GET /api/users/{id}` - Obter usuário por ID com pets e endereços
- `POST /api/users` - Criar novo usuário
- `PUT /api/users/{id}` - Atualizar perfil do usuário

### Pets
- `GET /api/users/{userId}/pets` - Listar pets de um usuário
- `POST /api/users/{userId}/pets` - Adicionar novo pet ao usuário
- `PUT /api/pets/{id}` - Editar dados do pet
- `DELETE /api/pets/{id}` - Deletar pet (soft delete com flag IsDeleted)

### PetShops
- `GET /api/petshops` - Listar petshops próximos com filtro de distância
- `GET /api/petshops/{id}` - Detalhes completo do petshop + serviços + horários
- `GET /api/petshops/{id}/services` - Listar serviços de um petshop
- `GET /api/petshops/search` - Buscar petshops por nome, tipo, com paginação

### Services
- `GET /api/services/{id}` - Detalhes do serviço + preço + sub-serviços
- `GET /api/services/{id}/availability` - Horários disponíveis para agendamento
- `POST /api/petshops/{petshopId}/services` - Criar novo serviço no petshop (admin)

### Appointments
- `GET /api/users/{userId}/appointments` - Listar agendamentos do usuário (com filtro por status)
- `POST /api/appointments` - Criar novo agendamento
- `PUT /api/appointments/{id}` - Atualizar agendamento (data/hora/notas)
- `PUT /api/appointments/{id}/cancel` - Cancelar agendamento
- `PUT /api/appointments/{id}/confirm` - Confirmar presença no agendamento

### Addresses
- `GET /api/users/{userId}/addresses` - Listar endereços do usuário
- `POST /api/users/{userId}/addresses` - Adicionar novo endereço

## BACKEND (.NET) - Padrão Controller/Service/Repository

```csharp
// Controller
[ApiController]
[Route("api/[controller]")]
public class ResourceController : ControllerBase
{
    private readonly IResourceService _service;
    public ResourceController(IResourceService service) => _service = service;
    
    [HttpGet]
    public async Task<ActionResult<IEnumerable<ResourceViewModel>>> GetAll()
    {
        var resources = await _service.GetAllAsync();
        return Ok(resources);
    }
}

// Service
public class ResourceService : IResourceService
{
    private readonly IResourceRepository _repository;
    public ResourceService(IResourceRepository repository) => _repository = repository;
    
    public async Task<IEnumerable<ResourceViewModel>> GetAllAsync()
    {
        return await _repository.GetAllAsync();
    }
}
```
